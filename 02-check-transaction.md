# Check Transaction API

Retrieves the current status of a transaction created within the last 7 days. For older transactions, use the [Get Transaction Details](./03-get-transaction-details.md) API.

## Endpoint

```
POST /api/payment-gateway/v1/payments/check-transaction-2
```

**Base URL**: `https://checkout-sandbox.payway.com.kh/` (sandbox) | `https://checkout.payway.com.kh/` (production)

**Content-Type**: `application/json` (Note: sandbox may require `multipart/form-data` despite this documentation)

**Rate Limit**: 600 requests/second

## Request Parameters

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `req_time` | string | Yes | Request date and time in UTC format as `YYYYMMDDHHmmss` |
| `merchant_id` | string | Yes | Unique merchant key provided by ABA Bank |
| `tran_id` | string | Yes | Your purchase transaction ID |
| `hash` | string | Yes | Base64-encoded HMAC-SHA512 hash |

## Hash Generation

```php
$api_key = "API KEY PROVIDED BY ABA BANK";
// Official docs show: $b4hash = $merchant_id . $tran_id;
// Actual sandbox requires req_time in the hash:
$b4hash = $req_time . $merchant_id . $tran_id;
$hash = base64_encode(hash_hmac('sha512', $b4hash, $api_key, true));
```

## Response

**HTTP 200**

### `data` Object

| Field | Type | Description |
|-------|------|-------------|
| `payment_status_code` | integer | `0` = APPROVED/PRE-AUTH, `2` = PENDING, `3` = DECLINED, `4` = REFUNDED, `7` = CANCELLED |
| `payment_status` | string | `APPROVED`, `PRE-AUTH`, `REFUNDED`, `PENDING`, `DECLINED`, or `CANCELLED` |
| `total_amount` | number | Amount customer pays after discount |
| `original_amount` | number | Original purchase amount |
| `refund_amount` | number | Total of all refunded amount |
| `discount_amount` | number | Discounted amount (follows original transaction currency) |
| `payment_amount` | number | Amount customer has paid |
| `payment_currency` | string | Currency customer used to pay |
| `apv` | string | Transaction approval code |
| `transaction_date` | string | Date and time of transaction creation |

### `status` Object

| Field | Type | Description |
|-------|------|-------------|
| `code` | string | Response code |
| `message` | string | Details based on code |
| `tran_id` | string | Unique request ID auto-generated by payment gateway |

## Status Codes

| Code | Description |
|------|-------------|
| `00` | Success |
| `5` | Invalid hash |
| `6` | Transaction not found |
| `8` | Invalid merchant profile |
| `11` | Internal server error |
| `429` | Request limit reached |
